# youtube-uploader Guide

Everything you need to know — setup, usage, automation, and internals — in one place.

---

## What It Is

A Rust CLI and library for uploading videos to YouTube via the Data API v3. **You run it, it uploads, it exits.** No daemon, no background process, no service. Every invocation is a one-shot process.

Persistent state between runs lives on disk only:

| Path | Purpose |
|------|---------|
| `~/.config/youtube-uploader/credentials.enc` | AES-256-GCM encrypted OAuth2 tokens |
| `~/.config/youtube-uploader/profiles/*.toml` | Upload presets |
| `~/.config/youtube-uploader/resume/` | In-progress upload state (crash recovery) |

## Setup

### 1. Google Cloud Project

1. Go to [Google Cloud Console](https://console.cloud.google.com/apis/credentials)
2. Create an OAuth 2.0 Client ID — **Desktop app** type (or TV/Limited Input for device code flow)
3. Enable the **YouTube Data API v3**
4. Note your `client_id` and `client_secret`

### 2. Authenticate

```bash
# Set credentials (one-time)
youtube-uploader auth --client-id YOUR_ID --client-secret YOUR_SECRET

# Or use .env file with YOUTUBE_CLIENT_ID / YOUTUBE_CLIENT_SECRET
youtube-uploader auth
```

Follow the on-screen instructions — open a URL, enter a device code. The CLI waits for authorization and stores the tokens encrypted.

### 3. Multiple Channels

```bash
youtube-uploader auth              # → workspace "youtube" (default)
youtube-uploader -w cooking auth   # → workspace "cooking"
```

Switch Google accounts in the browser before completing the device code flow for brand accounts.

## CLI Reference

### Upload

```bash
# Basic upload (private by default)
youtube-uploader upload --file video.mp4 --title "My Video"

# With all options
youtube-uploader upload \
  --file video.mp4 \
  --title "My Video" \
  --description "Description here" \
  --tags "rust,programming" \
  --visibility private \
  --category 22 \
  --language en \
  --license youtube \
  --made-for-kids false \
  --contains-synthetic-media false \
  --embeddable true \
  --public-stats-viewable false \
  --publish-at "2026-05-20T12:00:00Z" \
  --description-suffix "\n\nUploaded by bot" \
  --recording-date "2026-05-18" \
  --profile default \
  --meta video.meta.toml \
  --workspace gaming
```

### Batch Upload

```bash
youtube-uploader batch manifest.csv --concurrency 2
```

CSV format:

```csv
file,title,workspace,profile,description,tags,visibility
vid1.mp4,Title 1,gaming,gaming,Desc,rust|code,public
vid2.mp4,Title 2,,,,private
```

Required columns: `file`, `title`. All others optional.

### Other Commands

```bash
# Dry run (show what would upload, don't upload)
youtube-uploader batch manifest.csv --dry-run

# List workspaces
youtube-uploader list

# Channel info
youtube-uploader channel

# Profile management
youtube-uploader profile list
youtube-uploader profile show default
youtube-uploader profile remove old-profile

# Workspace management
youtube-uploader workspace default cooking
```

### Global Flags

| Flag | Short | Purpose |
|------|-------|---------|
| `--workspace` | `-w` | Select workspace (channel) |
| `--profile` | `-P` | Select upload profile |
| `--output` | | `human` (default) or `json` |
| `--passphrase` | | Encryption passphrase |
| `--passphrase-file` | | Read passphrase from file |
| `--version` | | Print version |

## Upload Profiles

Named TOML presets in `~/.config/youtube-uploader/profiles/`:

```toml
# ~/.config/youtube-uploader/profiles/gaming.toml
visibility = "private"
category = "20"
made_for_kids = false
license = "youtube"
language = "en"
tags = ["gaming", "lets-play"]
description_suffix = "\n\nStreaming Tue/Thu 7pm"
```

All fields optional — only set what you want to default.

```bash
youtube-uploader -P gaming upload --file gameplay.mp4 --title "Stream"
# CLI flags override: still private even if profile says public
youtube-uploader -P gaming upload --file vid.mp4 --title "Stream" --visibility public
```

## Per-Video Metadata (`.meta.toml`)

For AI/automation pipelines, write a metadata file next to your video:

```toml
# video.meta.toml
title = "Automated Upload"
description = "Generated by AI pipeline"
tags = ["automated", "pipeline"]
category = "22"
visibility = "unlisted"
profile = "default"   # optional: loads profile first
```

```bash
# Auto-discovered: video.meta.toml next to video.mp4
youtube-uploader upload --file video.mp4

# Explicit path
youtube-uploader upload --file video.mp4 --meta /path/to/custom.meta.toml
```

### Resolution Order

Highest priority wins:

```
CLI flags  >  meta TOML  >  profile TOML  >  built-in defaults (private)
```

Tags are **merged** (profile + meta), not replaced.

## AI Automation Pipeline

```bash
# 1. Write metadata
cat > video.meta.toml << 'EOF'
title = "Auto Upload"
description = "Pipeline-generated"
tags = ["automated"]
made_for_kids = false
EOF

# 2. Upload, capture JSON result
youtube-uploader --output json upload --file video.mp4
# → {"workspace":"youtube","video_id":"dQw4w9WgXcQ","url":"https://...","title":"Auto Upload"}

# 3. Parse result in script
VIDEO_ID=$(youtube-uploader --output json upload --file video.mp4 | jq -r .video_id)
echo "Uploaded: $VIDEO_ID"
```

## Architecture

```
youtube-uploader/               # Library crate
├── src/
│   ├── lib.rs                 # Public exports
│   ├── youtube.rs             # YouTubeUploader (resumable upload, token refresh, delete)
│   ├── upload.rs             # VideoUpload, UploadResult, Visibility, License
│   ├── config.rs             # Encrypted CredentialStore with workspaces
│   ├── profile.rs            # UploadProfile, VideoMeta
│   ├── error.rs              # UploadError enum
│   ├── validation.rs         # File validation (size, extension, title)
│   ├── progress.rs           # ProgressListener trait + StderrProgressListener
│   ├── net.rs                # HTTP client, retry logic, SSRF protection
│   ├── resume.rs             # UploadState (crash recovery)
│   └── auth/
│       ├── mod.rs            # Shared TokenResponse
│       ├── device_code.rs    # OAuth2 device code flow
│       ├── auth_code.rs      # Browser authorization code flow (fallback)
│       └── refresh_token.rs  # Token refresh

youtube-uploader-cli/          # Binary crate (run-and-exit)
├── src/
│   ├── main.rs               # CLI entry point
│   └── output.rs             # Pretty-print output (all user-facing formatting)
└── tests/
    └── cli.rs
```

### Upload Flow

```
CLI parse → load credentials → refresh access token
→ initiate resumable upload → upload chunks (with retry)
→ complete → print result → EXIT
```

If the process crashes mid-upload, resume state is saved in `~/.config/youtube-uploader/resume/`. Call `resume()` to continue.

## Security

| Layer | Detail |
|-------|--------|
| Encryption | AES-256-GCM |
| Key derivation | PBKDF2, 100K iterations |
| Memory | `Zeroizing<String>` for all secrets, zeroize on drop |
| SSRF | Upload URLs validated to `*.googleapis.com` / `*.google.com` only |
| Token storage | Single encrypted file, passphrase-protected |
| Default visibility | **Private** — uploads never accidentally public |

## YouTube API Quotas

| Resource | Cost |
|----------|------|
| Upload (per video) | ~1,600 units |
| Daily quota | 10,000 units/project |
| Effective uploads/day | ~6 per project |
| Quota shared across | All workspaces using same OAuth2 client |

Quota resets daily at midnight Pacific Time.
